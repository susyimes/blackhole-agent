## Hypothesis

Skill-route discovery should preserve file-layout and metadata evidence before
proposal selection. COMPASS-style skill repositories can often be recognized by
`skills/.../SKILL.md` and skill registry metadata, FableCodex-style workflow
repositories by plugin/test/template shape, and game/frontend skill repositories
by director skill, scaffold, and QA path names. These signals should produce
only disabled local lanes: documentation, config, test, or code_patch.

## Change

- Added body-free `observed_paths` and `metadata_files` intake to
  `ExternalSkillRepositorySummary`.
- Added layout and metadata signal tags to disabled candidate records and
  proposal lane maps when present.
- Mapped layout signals only to bounded local lanes and kept runtime action,
  external skill activation, install, execute, clone, scaffold, and provider
  launch denied.
- Added a regression that classifies COMPASS, FableCodex, and Three.js-style
  file layouts while ignoring a generic `AGENTS.md` repository.
