## Change

Added a profile-aware validation manifest to the existing skill-route activation
surfaces:

- `local_activation_targets`
- `pass1_validation_matrix`
- `next_validation_step`

Each manifest row records the route profile, validation gate, required proof,
selected local lane, evidence basis, selected item IDs when present, and a
hashed replay command. The manifest keeps runtime action, external skill
activation, external harness execution, provider launch, remote execution,
profile writes, memory writes, raw URL export, raw target export, raw upstream
body export, and raw replay command export disabled.

Literal replay commands remain on established local replay surfaces for
operator execution. The new manifest is hash-only for replay command evidence.
