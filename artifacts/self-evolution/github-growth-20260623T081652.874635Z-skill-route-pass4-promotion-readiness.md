## Hypothesis

The slice already had lane maps, activation targets, pass handoffs, and
completion reports. The missing final-pass behavior was a compact promotion
readiness view on the core adoption manifest itself, so operators can see the
rollback-backed evidence requirements and replay commands before supervisor
handoff without traversing deeper completion panels.

## Change

Added `promotion_readiness` to `skill_route_discovery_adoption_manifest`.
It aggregates ready manifest rows into:

- selected bounded local lanes
- unique replay commands
- required promotion evidence, including rollback ref and rollback artifact
- hashed target paths from lane promotion proofs
- external-supervisor-only handoff metadata
- repeated denials for runtime action, external skill activation, external
  harness execution, provider launch, remote execution, and raw export

Each adoption manifest row now also carries its lane-specific
`promotion_proof`.
