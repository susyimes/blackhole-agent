## Evidence And Hypothesis

The active proposals were review-only for privacy-leakage-sensitive UI and
approval migrations, but the Omnigent mock-LLM e2e PR evidence showed a useful
local lesson: deterministic provider/mock migration needs an operator-visible
way to distinguish missing mock/provider wiring from misleading timeout or
startup failures.

Hypothesis: provider/runtime preflight control is more useful at the completion
of this capability slice if multiple preflight cases can be summarized into
body-free recovery hints. A scheduler or reviewer can then see which provider
routes are blocked, which are merely degraded, and which local fix is safe to
try next without reading raw URLs, CLI paths, environment keys, token values,
prompts, or diagnostics bodies.
