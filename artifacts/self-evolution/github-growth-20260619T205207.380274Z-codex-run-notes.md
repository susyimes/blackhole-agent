  provider credential pools.
- The credential-pool failover portion crosses the privacy boundary because

  sensitive. This run kept failover review-only and implemented only local,
  body-free diagnostics.
- Reviewed the named skill-route repositories only enough to confirm they do
  not displace the active provider/runtime theme for this pass.

## Hypothesis

Provider runtime preflight should classify 429 and exhausted usage-window
signals before retry or launch, produce recovery hints for operators, and prove
that response bodies, raw headers, credential labels, and credential values are
not exported. This gives the supervisor a replayable diagnostic without
implementing credential-pool failover.
