## Hypothesis

The Omnigent issue reports a Claude-native second-message timeout when prompt
readiness scans only a small terminal tail and a multi-line Claude Code status
footer pushes the prompt outside that window. The local reusable lesson is a
metadata-only provider preflight lane that proves the configured scan window
will see the prompt before a live provider session is launched.

## Changes

- Extended `provider_runtime_preflight` with a body-free `prompt_scan` model.
- Added diagnostics for a Claude prompt that sits beyond the legacy scan tail
  but inside the configured scan window.
- Added a blocking `prompt_scan_timeout_risk` failure mode when the configured
  scan tail cannot reach the prompt above a rendered status footer.
- Added a fixture for long Claude status output where legacy tail-5 scanning
  would time out but the configured tail-12 scan succeeds.
- Documented prompt scan assumptions, timeout symptoms, and the focused local
  validation command.
