## Evidence And Hypothesis

Omnigent PR evidence for REPL approval/mock output polling suggests a reusable local lesson: approval-surfacing tests should not depend on one immediate transcript sample. The existing local harness already models native approval paths, so the smallest useful behavior change is a deterministic, body-free polling contract for recorded approval output samples.

Hypothesis: adding `native_tool_policy.approval_output_poll` to `mock_llm_workflow_route` gives future REPL approval fixtures a bounded local validation lane. It should pass only when the expected approval marker appears within the recorded samples allowed by timeout/interval metadata, and it should continue to omit raw commands, session IDs, tool arguments, and output bodies.
