## Evidence

- Primary proposal: `p1-provider-config-multimodal-preflight`
- Evidence URL: https://github.com/omnigent-ai/omnigent/issues/515
- Reusable lesson: a vision-capable route can silently lose images when model metadata omits image input support or when multimodal content blocks are JSON-encoded into text instead of sent as native image payloads.

## Hypothesis

The local deterministic harness should fail before execution when a provider/model route receives image content but the model input metadata does not declare image support, or when multimodal blocks have already been flattened into a JSON text string. This makes future provider regressions observable without live provider calls or prompt-body export.
