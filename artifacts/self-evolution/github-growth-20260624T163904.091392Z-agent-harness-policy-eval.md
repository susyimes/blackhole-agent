## Evidence And Hypothesis

The carried Omnigent evidence describes public meta-harness behavior with policy,
sandbox, approval, and harness execution pressure. The local hypothesis is not
to execute Omnigent or import its implementation. The reusable lesson is a
bounded pre-execution lane: before a local harness capability can be treated as
runnable, policy decisions must be inspectable, ordered before the action, and
able to fail closed without leaking raw commands, provider config, credentials,
policy bodies, or upstream bodies.

## Change

Added `agent_harness_policy_eval` to the local harness evaluator. The lane
accepts body-free policy decisions and harness action plans, then reports:

- whether each action had a policy decision with an earlier sequence index
- whether the policy outcome allowed, blocked, required ASK/review, or was
  unknown
- whether an action kind is local-only or forbidden
- whether execution stayed unattempted

Allowed local action kinds are fixture/controller/report/mock/local-validation
shapes only. External harness execution, provider launch, remote execution,
credential access, and upstream agent activation remain forbidden.

## Review Notes

This lane is dry-run validation only. It can report a local fixture action as
policy-allowed after checks, but it still records `execution_attempted: false`.
Credential-pool failover, privacy-leakage behavior, live provider launch, live
external harness execution, push, promotion, and restart remain outside this
change.
