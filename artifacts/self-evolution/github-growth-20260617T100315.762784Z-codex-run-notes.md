## Hypothesis

The existing local skill router already ranks configured trigger terms above topical matches, but it can miss explicit skill-name invocation when the descriptor has no separate trigger term. Treating a skill's own name, plus `$name` and `@name` forms, as an explicit match makes routing more faithful to current public skill ecosystems without reading private skill bodies or exposing local data.
