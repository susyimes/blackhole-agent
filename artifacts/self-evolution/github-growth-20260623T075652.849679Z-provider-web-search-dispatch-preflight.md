## Hypothesis

The existing `provider_runtime_preflight` lane is the right local activation gate for this evidence. A metadata-only tool-dispatch check can block a non-OpenAI `web_search` route before runner launch unless the fixture supplies either registered local dispatch-handler evidence or replay evidence. This improves the skill-route/provider preflight slice without executing external tools or exposing tool arguments.

## Change

- Added `tool_dispatch` metadata to `provider_runtime_preflight`.
- Blocked non-OpenAI `web_search` without a registered or replayed dispatch handler as `provider_tool_dispatch_missing`.
- Preserved OpenAI native web-search passthrough.
- Added recovery hints for supervisor/operator replay.
- Added an inline regression, a durable local harness fixture, and an architecture note.
