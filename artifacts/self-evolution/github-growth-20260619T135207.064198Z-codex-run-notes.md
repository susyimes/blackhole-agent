## Evidence And Hypothesis

The active window emphasized turning provider and runtime configuration problems
into body-free diagnostics, recovery hints, and locally replayable validation.
The prior pass already rejected placeholder provider credentials before launch.

Hypothesis: provider/runtime preflight is more operator-visible when each
failure also emits a structured recovery hint with a stable code, scope, safe
action text, and env-var names only. This lets a supervisor or scheduler display
the next local fix without reading secret values, raw URLs, profile values, or
credential paths.
