## Hypothesis

Mixed agent/Codex/skill/workflow evidence should be visible as a bounded local lane before activation. If the harness only exposes generic term diagnostics, a supervisor has to infer whether the first route is `skill_route_discovery` or a secondary agent-harness lane. A compact body-free probe can make that decision inspectable while keeping secondary harness evaluation blocked until local corroboration exists.

## Change

Added `mixed_local_lane_probe` to `skill_route_discovery_lane` output. It is derived from already-normalized `candidate_lane_intake` rows and:

- selects `skill_route_discovery` as the primary route for mixed Codex + skill/skills + workflow evidence;
- treats agent/agents terms as the full mixed-signal marker;
- preserves only documentation, config, test, and code_patch proposal kinds;
- marks `agent_harness_eval_after_local_corroboration` as blocked;
- exports selected item IDs and hashes, not raw source URLs or upstream bodies;
- denies runtime action, external skill activation, external agent activation, external harness execution, provider launch, and remote execution.

Docs and doc-contract tests were updated to preserve the operator-facing behavior.
