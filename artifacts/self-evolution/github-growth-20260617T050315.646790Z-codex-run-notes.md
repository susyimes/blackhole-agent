Evidence summary:

Omnigent issue #444 reports that a top-level REPL can show `state: sleeping` while delegated sub-agents are still running. The reusable lesson is that parent liveness displays should aggregate active child work when the controller already has child-session metadata.

Hypothesis:

`latest-supervisor-heartbeat.json` is the local controller-facing status surface. Extending its derived `runner_liveness` to include active child-session counts and statuses makes reconnect/status views less misleading without changing scheduling, permissions, or child execution.
