## Evidence

- `https://github.com/omnigent-ai/omnigent`
- `https://github.com/omnigent-ai/omnigent/pull/629`

The PR describes an ambient OpenAI routing bug where `OPENAI_API_KEY` discovery ignored a companion `OPENAI_BASE_URL`, sending a gateway token to the default OpenAI endpoint. The reusable local lesson is to make ambient OpenAI token and base URL discovery explicit in provider preflight artifacts while keeping token and URL values out of logs and serialized diagnostics.

## Hypothesis

Codex mutation preflight should distinguish these local startup states before `codex exec` runs:

- neither `OPENAI_API_KEY` nor `OPENAI_BASE_URL` present
- only `OPENAI_API_KEY` present
- only `OPENAI_BASE_URL` present
- both present, implying an OpenAI-compatible gateway route

Recording only env-name presence, endpoint source, and value-redaction booleans improves future diagnosis of gateway routing mistakes without adding live provider calls or exposing secrets.
