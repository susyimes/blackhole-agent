## Hypothesis

Single-file agent configs can declare valid function tools that still disappear before execution if config parsing or routing drops
tool type, callable, or parameter schema metadata. Preserving those fields in local descriptors and validating the executable
registry reduces silent tool-surface loss.

## Changes

- Added optional `tool_type` and `callable_path` metadata to `ToolDescriptor` compatibility keys and call metadata.
- Added `load_single_file_agent_tool_descriptors`, `tool_descriptors_from_agent_config`, and `executable_tool_registry`.
- Added a compact single-file YAML parser fallback so this path is locally testable without a new PyYAML runtime dependency.
- Added a regression fixture and test asserting a `type: function` tool reaches the executable registry with dotted callable and
  JSON schema preserved.
