## Hypothesis

Provider/runtime preflight problems should surface as body-free diagnostics with
operator-visible recovery and replay state. The existing
`provider_runtime_recovery_summary` lane already aggregated hint codes, but did
not give the supervisor a compact readiness decision for blocked versus
degraded-only preflights.

## Change

Added `supervisor_readiness` to `provider_runtime_recovery_summary`. The summary
is derived only from preflight counts, blocked failure modes, recovery hint
codes, and local replay commands. It keeps provider runtime launch and remote
execution explicitly denied while allowing degraded-only mock/local replay to be
distinguished from blocked provider startup.

Updated architecture documentation for the new handoff field.
