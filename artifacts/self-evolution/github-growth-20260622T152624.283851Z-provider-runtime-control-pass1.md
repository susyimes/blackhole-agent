## Evidence And Hypothesis

The active window carried Omnigent PR #884 as evidence for configurable provider auth identity headers. The safe reusable lesson is not to handle token or user identity values locally; it is to make ambiguous trusted-header configuration visible before provider launch.

Hypothesis: adding a metadata-only provider runtime preflight lane for configurable auth headers will turn default-plus-custom trusted-header drift into body-free diagnostics and replayable recovery hints without exposing header names, header values, env names, credentials, private chats, PII, or personal data.

## Change Summary

- Added an `auth_header` subcheck to `provider_runtime_preflight`.
- Blocked missing, malformed, or ambiguous trusted auth-header configuration before runner invocation.

- Documented the body-free auth-header preflight surface in `docs/architecture.md`.
