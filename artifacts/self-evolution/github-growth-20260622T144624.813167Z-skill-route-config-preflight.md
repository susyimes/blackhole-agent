## Hypothesis

Skill-route discovery already constrained proposal kinds to documentation,
config, test, and code_patch. The missing operator-visible behavior for this
pass was a pre-implementation gate that records which bounded local lane is
selected for each selected skill/workflow evidence item, which lanes remain
queued, and whether truncated or unsupported evidence is blocked before any
implementation route proceeds.

## Change

Added `skill_route_implementation_preflight` to the route-hint lane map and
policy preflight. The preflight:

- selects a bounded local lane from already classified skill-route evidence;
- queues the remaining bounded lanes without adding new lanes;
- blocks unsupported lane drift before implementation;
- records selected and truncated digest item IDs;
- states that truncated item IDs are not allowed as `evidence_refs`;
- keeps runtime action, external skill activation, external harness execution,
  provider launch, remote execution, raw source URL export, and upstream body
  export denied.

The self-model was read and left unchanged because it already matched the run:
prefer locally validated behavior changes when rollback-backed, while keeping
offensive behavior and privacy leakage review-only.
