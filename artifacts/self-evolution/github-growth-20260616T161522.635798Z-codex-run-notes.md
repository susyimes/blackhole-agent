The issue describes a gap in comparing the same agent across harness/model variants on quality, cost, latency, tool calls,
and failure mode. It also implies that run traces and usage data may contain enough material for comparison, but the
local privacy boundary requires reports to avoid storing private prompt or output bodies by default.

## Hypothesis

A local, body-free harness comparison report improves blackhole-agent's validation and observability loop without crossing
the privacy-leakage review boundary. A useful first implementation can summarize existing JSON run artifacts into hashes,
metrics, and aggregate variant rows, leaving raw prompt/output inspection outside the default artifact.
