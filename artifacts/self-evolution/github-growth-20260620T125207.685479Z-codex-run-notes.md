public skill repositories should be converted into bounded, body-free local
candidate lanes before any activation decision. COMPASS-style skill ecosystems
add state/profile/routing pressure, Three.js skill packs add domain/director
and QA pressure, and FableCodex adds Codex workflow-gate pressure. None of this
evidence justifies installing, executing, cloning, scaffolding, or importing
external skill code during discovery.

## Hypothesis

The skill-route lane already has expanded lane matrices, activation manifests,
and profile review. Pass 2 benefits from a narrower preactivation intake panel:
one candidate-level view that lets a supervisor pick a bounded validation lane
without rehydrating raw source URLs or upstream bodies.
