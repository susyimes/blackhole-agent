## Hypothesis

Skill-route discovery already creates disabled registries and local proposal
lanes, but activation rows are the controller-facing object that a supervisor
would consume. Adding a pre-activation trust-boundary preflight over those rows
should make route activation more robust against future drift: external skill
activation, external harness execution, runtime actions, missing local
validation, or unsupported proposal kinds should fail before a lane is treated
as ready.
