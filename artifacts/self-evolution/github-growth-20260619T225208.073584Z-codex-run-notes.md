## Evidence

- Reviewed https://github.com/omnigent-ai/omnigent/issues/864.
- Reusable lesson: a controller must not surface a successful connected state when host registration fails after handshake because the host id belongs to a different account. The local analogue should block before activation, keep account details body-free, and provide a recovery hint.

## Hypothesis

Extending the existing `agent_harness_provider_registration` lane with host-registration state checks gives the supervisor a locally replayable way to catch misleading success states without launching a provider or exporting host/account identifiers.
