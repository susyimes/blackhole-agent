## Evidence

Omnigent PR #1109 describes native Windows support in core/degraded mode. The reusable local lesson is that Windows-only runtime gaps should be detected before launch and expressed as clear, actionable degraded diagnostics when a supported degraded route exists. The upstream evidence also names classes of unavailable native support, including terminal, isolation, proxy, shell, environment, and process-management assumptions.

## Hypothesis

The local provider-runtime preflight should distinguish unsafe Windows runner metadata from explicit degraded-mode Windows support. If the fixture declares degraded mode and structural launch metadata is replayable, unavailable shell, dependency, or capability evidence should become a body-free `provider_windows_runner_degraded_mode` diagnostic with local replay only. Without that allowance, the same support gap should still block before launch.

## Changes

- Extended `evaluate_provider_windows_runner_preflight` with dependency and capability availability metadata.

- Preserved blockers for malformed commands, unquoted paths, unresolved or out-of-repo workspaces, and missing local replay proof.
