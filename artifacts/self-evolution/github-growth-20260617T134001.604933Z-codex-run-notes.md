## Hypothesis

Omnigent presents a meta-harness pattern with multiple provider routes, policies, sandboxing, and shared agent sessions. The reusable local lesson is that provider/runtime preflight diagnostics should fail early while remaining safe to store and review. This repository already had token-value-safe preflight checks, but malformed operator input for `--token-env` could still be echoed into diagnostics. Treating invalid env var text as untrusted closes that gap without changing the review-only privacy boundary.

## Change

- Removed raw `--token-env` echoing from provider preflight diagnostic strings.
- Suppressed invalid token env names from the structured preflight payload by recording `token_env_name: null` and `token_env_name_recorded: false`.
- Preserved valid env var names and boolean presence metadata for operator debugging.
- Added tests proving secret-shaped malformed token-env text is absent from both JSON preflight payloads and startup exception messages.
- Updated the README provider/token setup note.
