## Evidence And Hypothesis

The selected proposal asked for local validation of provider configuration preflight when `wire_api` is set to `chat`. The reusable lesson from the supplied Omnigent evidence is that chat-oriented provider routing should be explicit and locally exercised before provider launch; otherwise a configured chat route can be silently ignored by a generic provider runtime path.

Hypothesis: adding a metadata-only provider runtime preflight lane for `wire_api` improves future skill-route/provider proposals by turning chat-vs-responses route drift into a replayable blocked state with recovery hints, without launching providers or exporting raw config values.
