## Hypothesis

External skill repositories are useful as routing evidence, but discovery events
must remain classification-only until a separate local validation path promotes
them. Repository creation should not install or enable a skill, and repository
deletion should not delete or execute local state.

## Changes

- Extended `ExternalSkillRouteCandidate` with normalized discovery event kind,
  requested actions, blocked action diagnostics, sanitized candidate lanes, and
  record-only lifecycle effects.
- Added lifecycle fixtures for create/delete discovery events that request
  install, execute, local deletion, or an unsupported lane.
- Documented the lifecycle boundary in the upstream evidence interpretation
  contract.
