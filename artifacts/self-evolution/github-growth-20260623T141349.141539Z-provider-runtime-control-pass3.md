## Hypothesis

Omnigent-style meta-harness failures show provider/runtime problems that need
body-free, operator-visible diagnostics before activation. Existing local
preflights already covered startup `Errno 8` and proxy/auth env fallback. The
remaining useful lesson from issue evidence was runner turn-context desync:
orphaned callbacks after a harness disconnect and mid-turn buffered message,
with policy defaulting open during missing context.

## Change

- Added `runner_state` diagnostics to `provider_runtime_preflight`.
- Added blocker failure modes:
  - `provider_runner_turn_context_desync`
  - `provider_runner_policy_fail_open`
- Added recovery hints for active-turn rebind, self-heal/watchdog, and
  fail-closed policy handling before provider launch.
- Added a local Omnigent fixture that asserts no raw session ref, logs,
  callback bodies, policy bodies, raw source URLs, or provider runtime launch
  are exported.
- Documented the preflight boundary in `docs/skill-route-discovery.md`.
