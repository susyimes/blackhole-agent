## Evidence

- `https://github.com/omnigent-ai/omnigent` describes an agent meta-harness that routes Claude Code, Codex, Cursor, Pi, and custom agents through common orchestration.
- `https://github.com/omnigent-ai/omnigent/pull/837` is a CI review-model tuning PR. The reusable local lesson is that opaque review model IDs can fail at dispatch if the configured gateway does not resolve or support them, so review-model routes need preflight validation before review execution.

## Hypothesis

Provider runtime preflight should catch unavailable, unsupported, missing, or unexercised review-model routes as body-free metadata before any live review runs. The recovery summary should expose a stable hint code so a scheduler or operator can choose the next local fix without reading prompts, outputs, tokens, URLs, or raw model IDs.
