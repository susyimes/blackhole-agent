Reusable lesson: connector-native tool-call policy gates should fail closed. A
policy evaluator returning DENY, raising an error, or timing out must not expose
the tool as executable.

## Hypothesis

Adding an optional, injectable tool-call policy evaluator to local tool routing
will make future connector-native policy gates inspectable and fail-closed
without changing existing static routing behavior when no evaluator is present.

## Change

- Added `ToolCallPolicyResult`, `ToolCallPolicyEvaluator`, and a `denied` route
  for policy-gated tool descriptors.
- Routed policy DENY, evaluator exceptions, and evaluator timeouts to
  non-executable decisions with explicit reasons.
- Extended batch routing, preflight diagnostics, and executable registry
  construction to accept the optional evaluator.
- Added tests for ALLOW pass-through, explicit DENY, evaluator error, timeout
  simulation, registry exclusion, and preflight route counts.
