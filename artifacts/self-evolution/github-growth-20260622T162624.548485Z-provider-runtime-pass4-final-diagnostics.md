## Evidence And Hypothesis

The current provider-runtime-control window asks for body-free diagnostics, recovery hints, and locally replayable
validation before provider/runtime activation. Existing pass-4 completion already surfaced provider runtime handoff
state, but the final supervisor handoff did not expose one compact packet joining diagnostic panel readiness, sample
gate state, completion handoff state, recovery hints, replay commands, and explicit launch denial.

Hypothesis: a provider-runtime-only final diagnostics packet makes pass-4 completion easier for a supervisor to replay
without changing non-provider skill-route completion output or allowing provider/runtime launch from the kernel.
